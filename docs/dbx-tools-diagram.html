Auth Failover (Sticky)

Every API/CLI call goes through dbxExec() or dbxApi(). If the primary PAT profile fails with auth errors (401, 403, SCIM inactive, token expired), it automatically retries with the service principal profile. Once failover triggers, activeProfile stays on SP for the rest of the session.

Cleanup Order

dbx_cleanup deletes in dependency order: pipelines first (releases table ownership), then tables, then jobs, then dashboards. Prevents orphaned-reference errors.

Polling Tools

dbx_run_notebook and dbx_poll_pipeline both submit work and poll with configurable intervals + timeouts. They use onUpdate callbacks for real-time progress and respect signal.aborted for cancellation.

Dashboard Deploy

dbx_deploy_dashboard is create-or-update: checks for existing dashboard by name, then PATCH or POST. Always publishes with embed_credentials: false (required for personal MS account auth on this workspace).
