Why JDBC for SAP HANA specifically? SAP HANA is not a natively supported Lakehouse Federation query federation source (supported: MySQL, PostgreSQL, Oracle, SQL Server, Synapse, Teradata, BigQuery, Snowflake, Redshift, Salesforce, Databricks). The JDBC Unity Catalog Connection (Beta, DBR 17.3+) provides governed access, credential isolation, and Spark integration. For SAP BDC customers, the new SAP Databricks product (Feb 2025) with Delta Sharing is the preferred path — zero-copy, semantics-preserving, and fully governed.
Setup Steps
1
Upload SAP HANA JDBC Driver to a Volume
-- Create volume for JDBC drivers CREATE VOLUME IF NOT EXISTS my_catalog.drivers.jdbc_jars; -- Upload ngdbc.jar (SAP HANA JDBC driver) -- via UI: Catalog → Volume → Upload -- or via Databricks CLI: -- databricks fs cp ngdbc.jar \ -- dbfs:/Volumes/my_catalog/drivers/jdbc_jars/ -- Grant read access GRANT READ VOLUME ON VOLUME my_catalog.drivers.jdbc_jars TO `data-engineers`;
2
Create the JDBC Connection in Unity Catalog
DROP CONNECTION IF EXISTS sap_hana_prod; CREATE CONNECTION sap_hana_prod TYPE JDBC ENVIRONMENT ( java_dependencies '["/Volumes/my_catalog/drivers/jdbc_jars/ngdbc.jar"]' ) OPTIONS ( url 'jdbc:sap://sap-hana-host:30015/?currentSchema=MY_SCHEMA', user 'DATABRICKS_SVC', password '{{secrets/sap-scope/hana-pwd}}', externalOptionsAllowList 'dbtable,query,partitionColumn,lowerBound,upperBound,numPartitions' );
3
Grant Access & Verify
-- Grant USE to data team GRANT USE CONNECTION ON CONNECTION sap_hana_prod TO `data-engineers`; -- Verify connection details DESCRIBE CONNECTION sap_hana_prod;
4
Query SAP HANA from Databricks
# PySpark — read via Spark Data Source API df = ( spark.read.format('jdbc') .option('databricks.connection', 'sap_hana_prod') .option('dbtable', 'SAPABAP1.VBAK') # Sales orders .load() ) df.display() # SQL — use remote_query() # SELECT * FROM # remote_query('sap_hana_prod', # query => 'SELECT * FROM VBAK');
5
Persist to Delta for Performance (Optional)
# Write SAP data to Delta Lake (medallion) (df .write .format("delta") .mode("overwrite") .option("overwriteSchema", "true") .saveAsTable("bronze.sap.sales_orders") ) # Now query locally — no JDBC roundtrip # Use for dashboards, ML, joins with # other lakehouse data
6
Networking: VNet Peering to SAP HANA
# Ensure connectivity: # 1. SAP HANA must be reachable from # Databricks compute (VNet peering # or Private Link) # 2. Port 30015 (or 3NN15) must be open # 3. For serverless: configure firewall # for serverless compute access # 4. Test with connectionTest UDF: SELECT connectionTest('sap_hana_prod');

Databricks SA Angle

  • Position JDBC connection as zero-ETL for ad-hoc SAP access
  • Demo: query SAP HANA live from a Databricks notebook in 5 min
  • Highlight: credentials never exposed to end users (UC governance)
  • For higher volume / lower latency: recommend Lakeflow Connect or ADF
  • For SAP BDC: highlight Delta Sharing connector (zero-copy)
  • Competitive edge: Snowflake has no equivalent governed JDBC

Enterprise SA Angle

  • Store SAP HANA creds in Key Vault, reference via secrets scope
  • Create a dedicated service account (DATABRICKS_SVC) with read-only
  • VNet-inject the workspace; VNet-peer to SAP network segment
  • For production: persist to Delta (medallion) — don't query HANA live
  • Set up Lakeflow Jobs for nightly incremental loads
  • Audit all SAP access via Unity Catalog system tables
