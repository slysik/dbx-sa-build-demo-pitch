🏦Banking Churn Detection Platform

Real-Time Customer Retention Intelligence System

📊Project Overview:
A production-grade data pipeline that ingests real-time banking app events from Bank of America iOS users, detects customers at-risk of switching banks, and empowers customer service teams with actionable interventions. Built on Databricks with real-time streaming (Zerobus), AI/ML predictions, and interactive dashboards.

🎯Business Problem

Banks lose customers when engagement drops — failed logins, app crashes, lack of transactions, support calls. Traditional analytics show this AFTER they've left. We need real-time detection + immediate intervention.

System Architecture

BOA iOS App (2K events/batch) ↓ Zerobus gRPC (real-time stream) ↓ Bronze Layer (raw JSON, Zerobus Ingest) ↓ SDP Pipeline (serverless compute) ↓ Silver Layer (parsed, enriched with engagement metrics) ↓ SDP Pipeline ↓ Gold Layer (ML churn scores, intervention tracking) ↓ ┌─ Dashboard (interactive at-risk customer view) ├─ Genie Space (natural language Q&A) ├─ Web App (customer service UI for interventions) └─ SQL Alerts (high-risk escalations)

🔑Key Features

1️⃣ Real-Time Ingestion (Zerobus): 2,000 banking events pushed every 5 minutes. Zero latency, gRPC protocol, built-in acknowledgment.
2️⃣ Intelligent Risk Scoring: Rule-based (30s) + ML predictions (LogisticRegression). Scores combine RFM, engagement, and behavioral signals.
3️⃣ Engagement Metrics: App opens, transactions, failed logins, crashes, support calls. Weekly aggregation with risk signal counting.
4️⃣ Intervention Tracking: Log every outreach attempt (call, email, offer). Track what worked, measure retention impact.
5️⃣ Multi-Channel Delivery: Dashboard for monitoring, Genie for Q&A, Streamlit app for reps, SQL alerts for escalations.

📈Expected Outcomes

  • Earlier Detection: Catch at-risk customers within 5 min of warning signal (not weeks later)
  • Targeted Interventions: Rule engine recommends specific actions (tech support, discount, VIP call)
  • Measurable Impact: Track intervention success rate — did we retain them?
  • Data-Driven Decisions: Understand which interventions work best by segment
💡Business Impact: Reduce churn rate by 15-20% through early intervention, increase customer satisfaction by showing they're valued.

🏗️Medallion Layer Design

Bronze Layer: Raw Ingestion

Property Value
Table finserv.churn_demo.bronze_banking_events
Source Zerobus gRPC stream (BOA iOS app)
Schema event_id, user_id, event_type, timestamp, is_risk_signal, metadata_json, ingest_ts
Append Mode 100% append-only (immutable source of truth)
Event Types app_open (40%), transaction (30%), failed_login (10%), app_crash (8%), support_call (7%), slow_transaction (5%)
Volume 2,000 events per 5-min batch

Silver Layer: Enrichment

Two Materialized Views (SDP):

View Purpose Key Columns
silver_user_events Deduped, parsed raw events event_id, user_id, event_type, timestamp, is_risk_signal
silver_user_metrics Weekly engagement aggregation (RFM) user_id, metric_week, weekly_app_opens, weekly_transactions, weekly_risk_signals, weekly_failed_logins, weekly_crashes, weekly_support_calls, engagement_health

Gold Layer: ML-Ready + Tracking

View/Table Purpose Key Columns
gold_churn_risk Risk scores + action recommendations user_id, rule_based_score (0-100), risk_tier, engagement_health, recommended_action
gold_churn_predictions ML model predictions (LogisticRegression) user_id, ml_churn_probability (0-100), ml_churn_class
gold_interventions Intervention log + outcomes intervention_id, user_id, intervention_type, intervention_result, retained_after_intervention

📊Risk Scoring Logic

Rule-Based Scoring (Real-Time, 0-100)

RISK_SCORE = BASE_SIGNALS + ENGAGEMENT_PENALTY BASE_SIGNALS: - Days since last event > 30 days → +40 pts (DORMANT) - Risk signals ≥ 3 per week → +25 pts (multiple warnings) - Failed logins ≥ 2 per week → +15 pts (auth issues) - App crashes ≥ 2 per week → +10 pts (stability) - Support calls ≥ 1 per week → +10 pts (asking for help) ENGAGEMENT_PENALTY: - No app opens this week → +30 pts - App opens but no transactions → +20 pts

Risk Tiers

Tier Score Range Action
CRITICAL 75+ Win-back campaign + VIP support
HIGH 50-74 Tech support or incentive offer
MEDIUM 25-49 Monitor closely, optional check-in
LOW 0-24 Standard engagement

🤖ML Model: LogisticRegression

Training Schedule: Weekly (Monday 2 AM)
Features: RFM metrics + engagement signals (7 features)
Target: Binary classification (churn_risk_label: 0=safe, 1=at-risk)
Output: Churn probability (0-100%) per user
Tracking: MLflow (accuracy, AUC-ROC, AUC-PR logged)

🔄Data Flow Timeline

  1. T+0min: Producer generates 2K events, pushes to Zerobus
  2. T+1min: Bronze table appended (raw events)
  3. T+2min: SDP processes Silver (dedup, parse)
  4. T+3min: SDP computes Gold (risk scores, recommendations)
  5. T+4min: Dashboard refreshed, alerts evaluated
  6. T+5min: Next batch begins

💾Data Retention & Partitioning

  • Bronze: 90 days (audit trail, compliance)
  • Silver: 2 years (historical analysis)
  • Gold: 1 year (training data for future models)
  • Clustering: Liquid Clustering on user_id (not legacy partitioning)

⚡Quick Start (5 minutes)

  1. Validate Setup:
    databricks bundle validate
    Checks workspace, schema, warehouse.
  2. Deploy Bundle:
    databricks bundle deploy
    Creates jobs, pipelines, alerts, and dashboard in Databricks workspace.
  3. Run Producer (First Batch):
    databricks jobs run-now --job-id [produce_banking_events]
    Generates 2K events, pushes to Bronze table.
  4. Start SDP Pipeline:
    databricks pipelines start-update --pipeline-id [churn_pipeline]
    Processes Bronze → Silver → Gold (takes ~2 min).
  5. View Dashboard:
    databricks workspace list /Users/lysiak043@gmail.com/dashboards/
    Open "[dev churn] Real-Time Churn Detection" dashboard.
  6. Launch Intervention App:
    streamlit run src/app/churn_intervention_app.py
    Opens web UI for logging interventions.

🔧Configuration & Customization

Adjust Risk Thresholds

Edit src/pipeline/03_gold_churn_scores.sql:

-- Change CRITICAL threshold from 75 to 70 WHEN m.days_since_last_event > 30 THEN 40 -- ← Adjust these scores WHEN m.weekly_risk_signals >= 3 THEN 25

Change Event Batch Frequency

Edit databricks.yml:

schedule: quartz_cron_expression: "*/5 * * * *" # Every 5 min # Change to: "*/10 * * * *" for every 10 min # Or: "0 * * * *" for every hour

Retrain ML Model More Often

Edit databricks.yml:

schedule: quartz_cron_expression: "0 2 * * 1" # Weekly Monday 2am # Change to: "0 2 * * *" for daily at 2am

💡Production Tips

  • Scale: The system handles 2K events/5min today. To scale to 10K/min, just increase the RANGE() in the producer notebook and adjust SDP shuffle partitions.
  • Cost: All compute is serverless (no cluster needed). SDP takes ~90s per run. Costs ~$0.50/day for small scale.
  • Data Retention: Bronze kept 90 days for compliance. Archive to cheap storage after.
  • Monitoring: Check gold_churn_risk row count every 6 hours. Should grow steadily, not spike.
